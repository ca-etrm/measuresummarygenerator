Purpose

The POU technical reference manual (TRM) provides information on eligibility, energy and water savings, costs, and other impacts - as well as the methodology used to estimate those impacts - for energy efficiency measures and projects. The energy savings estimates are used to report program accomplishments and measure progress towards program goals.

Manual Organization

The TRM consists of both this document and the permutations summary spreadsheets embedded with this document.
The TRM presents all active deemed measure package versions hosted in the California Electronic Technical Reference Manual (eTRM). This section explains the sections and subsections within this manual.
Table of Contents: Lists all the measures included in this document and their effective date ranges. Measures are grouped by their Use Category, such as HVAC, Food Service, etc.
Measures:The pages for each measure listing present text and data in the following order (with screen capture examples below to illustrate the respective sections/subsections):
  • Measure Details: The start of a new measure page. Section includes the Measure Name, hyperlink to the measure in the eTRM, Use Category, Program Administrator (PA) Lead for the measure (i.e., the organization that oversaw the measure development), Measure Version ID, and Effective Date Range (i.e., Start Date and End Date) to indicate the measure's active period.
  • Measure Case and Base Case Definition: The two tables in this section provide high level description of the measure and define the determinants (i.e., energy efficient measure case condition, base case condition, and/or installation circumstances) that characterize the measure.
    The "Offering ID" table describes the scope of the energy efficient measure case. The "Base Case Description" describes the base case that is being replaced. Definitions of "Measure Offering Description", "Existing Description", and "Standard Description" can be found in the Key Terminology section of this document.
  • Applicable Parameters: Lists the parameter labels that are applicable for each measure. The definition of each parameter can be found in the Key Terminology section.
  • Average Impacts: Provides the measure-level averages of energy savings, costs, water savings, and useful life. Values are calculated by averaging the applicable eTRM permutations. For details on the calculation methodology used to derive the impact data, click the hyperlink to be directed to the appropriate characterization section in the eTRM.
  • Streamlined Permutations: Provide the same impact fields as above at the measure offering and building location (i.e., climate zone) level. Values are calculated by averaging the applicable eTRM permutations for each measure offering and climate zone. These are external Excel spreadsheets attached with the TRM, grouped by Use Category, and accessible via hyperlinks located at the end of each "Average Impacts" table.
  • Other: Includes text describing program exclusions, quality assurance related to design, installation, commissioning, and operation, and other important notes.
    Program Exclusions field highlights any rules or restrictions that limit the eligibility of the measure, such as excluded market segments and building types.
    Quality Assurance field lists the verification and quality-related activities required for the design, installation, commissioning, and operation of the measure.
    Important Notes field documents miscellaneous information important to the measure not already captured in other sections.
Key Terminology: Provides definitions and supplementary information for each field in the measure data specification and for other important terms used in this document.
Data Table: Refers to large data tables that are not fully included in the main sections of the TRM due to size and usability in PDF format. These tables include Effective Useful Life (EUL), Net to Gross (NTG), and Gross Savings Installation Adjustment (GSIA). Click on the embedded icon to open each Excel spreadsheet.
Appendix:
  • Permutations Summary Spreadsheet: All the summary permutation spreadsheets accessible from one central location.
  • Permutation Data Specification: An embedded Excel workbook that lists the data specification fields in the eTRM permutation table or the POU TRM streamlined permutations table, in the order that they appear. The permutations tables (i.e., original eTRM and POU TRM streamlined permutations) are listed in their respective tabs. The following columns describe each data specification fields:
    • [Field] Verbose Name Human-readable name of the data specification field. For the eTRM, verbose name is used on the eTRM website or in the permutation CSV file downloadable from eTRM measure page. For the POU TRM, it is used in the PDF and the "Pivot Summary" tab of the streamlined permutations file.
    • [Field] Reporting Name: API name of the data specification fields. This API "Reporting Name" is used in the API or Postgres SQL read-only database and the "Table Summary" tab of the POU TRM streamlined permutations file.
    • Order: Order in which the field appears in the permutation. List should be sequential.
    • Grouping: Type or category of the data specification fields.
    • Type:
      • How fields are generated in the permutation tables:
        • Automatically generated with measure metadata
        • Manually input and mapped by the measure developer
        • Uploaded data from external CET
      • How fields are derived from the eTRM counterpart fields:
        • Concatenation of >1 eTRM fields
        • Average of eTRM numerical values (eTRM permutations are collapsed into 1 POU TRM permutation)
        • Concatenation of eTRM string values (eTRM permutations are collapsed into 1 POU TRM permutation)
        • Calculated based on certain logics
    • Data type: Type of the values, such as "String" and "Decimal".
    • Sample value: Example value in this field.
    • Definition: Free text definition field for each data specification.
  • Sunsetted (or Deactivated Measures): List of sunsetted or deactivated measure versions plus three flag fields to indicate reasons for sunsetting or deactivating:
    • TRM Update: Measures from a previous TRM that are no longer included in the new TRM.
    • Measure Version Update: Measure version is expired or past its End Date and a new measure version with data updated for the latest program year is available. The latest version should be used.
    • Measure Sunsetted/Deactivated: The entire measure package is sunsetted or deactivated without a replacement version for various reasons, such as code change or low measure uptake, etc.
